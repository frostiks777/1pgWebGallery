# ✅ Чеклист после оптимизации

## 🎯 Что нужно сделать

### 1. Пересборка проекта
```bash
# Остановить dev сервер если запущен (Ctrl+C)

# Очистить старый кэш (опционально)
rm -rf .cache/images/*
rm -rf .cache/medium/*
rm -rf .next

# Пересобрать проект
npm run build
# или
bun run build

# Запустить
npm run start
# или  
bun run start
```

### 2. Проверка функциональности
- [ ] Откройте http://localhost:3000
- [ ] Проверьте что галерея загружается
- [ ] Переключите несколько layouts (Masonry, Bento, etc.)
- [ ] Откройте любое фото в Lightbox
- [ ] Проверьте навигацию в Lightbox (← →)
- [ ] Попробуйте скачать фото (кнопка Download)
- [ ] Проверьте сортировку (по имени, по дате)

### 3. Проверка производительности

#### В Chrome DevTools:
1. **Network Tab:**
   - [ ] Откройте DevTools (F12) → Network
   - [ ] Отфильтруйте по Img
   - [ ] Обновите страницу (F5)
   - [ ] Проверьте что все изображения:
     - В формате WebP
     - Размер thumbnail ~50-150 KB
     - Есть заголовок `Cache-Control`
   - [ ] Обновите еще раз - должны быть кэшированные запросы (304 или from disk cache)

2. **Performance Tab:**
   - [ ] Performance → Record
   - [ ] Прокрутите галерею
   - [ ] Stop
   - [ ] Проверьте что нет долгих задач (> 50ms)

3. **Lighthouse:**
   - [ ] Lighthouse → Performance
   - [ ] Generate report
   - [ ] Score должен быть > 85 (лучше > 90)

### 4. Проверка кэша
```bash
# После загрузки галереи, проверьте что создается кэш
ls -la .cache/images/

# Должны быть файлы вида:
# {hash}.webp
```

## 📊 Ожидаемые результаты

/api/images?path=...&size=thumbnail

## 🐛 Если что-то не работает

### Изображения не загружаются
```bash
# Проверить что Sharp установлен
npm list sharp

# Переустановить если нужно
npm install sharp --save

# Проверить права на .cache
chmod -R 755 .cache/
```

### Ошибки TypeScript
```bash
# Очистить и пересобрать
rm -rf .next/
npm run build
```

### Старые изображения в кэше
```bash
# Полностью очистить кэш
rm -rf .cache/images/*
# Перезапустить сервер
```

### WebDAV не работает
1. Проверить `.env.local`:
   - WEBDAV_URL
   - WEBDAV_USERNAME  
   - WEBDAV_PASSWORD
   - PHOTOS_DIR
2. Проверить логи в консоли сервера
3. Попробовать `/api/images?path=...&size=thumbnail` в браузере
